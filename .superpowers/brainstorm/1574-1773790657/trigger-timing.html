When should the figure be generated?

This shapes the whole architecture — pick the trigger that fits how you study

A

On-Demand — "Generate Figure" button

A button appears below the question text when no image exists. Tap it → AI generates the diagram in ~3s → it appears and is cached. Never fires automatically, zero cost until clicked.

Pros

  • Zero API cost until you want it
  • You stay in control
  • Cached forever after first tap

Cons

  • Requires manual tap every new question
B

Auto — generates silently when question loads

As soon as a question with no image is shown, a background AI call fires. A small spinner appears where the figure will be. Diagram slots in when ready. Cached after first view.

Pros

  • Seamless — figure just appears
  • No extra interaction needed

Cons

  • API call on every unimaged question, even if you skip it
  • Slight distraction during load
C

Batch Pre-generate — run once, embedded in all questions

A one-time script generates figures for all 179 image-less questions, writes the HTML into storage, done. Every question instantly shows its figure from then on — no per-session AI calls.

Pros

  • Fastest in-session (no wait ever)
  • Works offline after generation

Cons

  • One upfront API cost (~179 calls)
  • Requires running a script once
